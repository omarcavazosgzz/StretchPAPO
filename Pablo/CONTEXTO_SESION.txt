1) VISION DEL PROYECTO (lo que quiero lograr)
--------------------------------------------------------------------------------
Que el robot Stretch, dentro de RoboCasa, abra cajones y agarre objetos para
llevarlos a un punto de interes (la estufa), usando SOLO informacion de las
camaras para detectar/ubicar. Ir paso por paso, verificando cada fase. Sin chocar.

Fases:
  Fase 1a: la camara de la CABEZA busca, se acerca y centra el objeto.
  Fase 1.5: posicionarse (PARALELO, sin chocar) y localizar el objeto POR CAMARA.
  Fase 1b: la camara del BRAZO (muneca) centra el objeto sin que la cabeza lo pierda.
  Fase 2 : agarrar el objeto.
  Fase 3 : llevarlo cerca de la estufa (u otra zona) + abrir cajones. Anti-colision.

3) ENTORNO (Windows, portable a Ubuntu)
--------------------------------------------------------------------------------
uv (C:\Users\<user>\.local\bin) y git (C:\Program Files\Git\cmd) deben estar en el
PATH (abrir PowerShell NUEVA tras instalarlos). Montaje desde C:\dev\StretchPAPO:
    uv sync
    uv pip install -e ".[robocasa]"
    uv pip install -e "robocasa@third_party/robocasa"
    uv pip install -e "robosuite@third_party/robosuite"
    uv run third_party/robosuite/robosuite/scripts/setup_macros.py
    uv run third_party/robocasa/robocasa/scripts/setup_macros.py
    "y" | uv run third_party/robocasa/robocasa/scripts/download_kitchen_assets.py
(submodulos FORKS: hello-binit/robocasa, SilentSammy/robosuite; si faltan:
    git submodule update --init --recursive )
Windows: NO usar MUJOCO_GL=egl (eso es Linux). multiprocessing 'spawn' -> el codigo
ejecutable va dentro de if __name__ == "__main__".

4) ARCHIVOS (en Pablo/)
--------------------------------------------------------------------------------
scene.py        : cocina custom + inyecta los objetos garantizados + limpia mostrador.
detection.py    : OracleDetector (proyeccion 3D->2D, ~2px) + YoloDetector (stub).
control.py      : PosServo = control por POSICION (sim.move_to_many). (El control por
                  velocidad del toolkit NO movia 2 juntas a la vez; bug arreglado.)
sim_setup.py    : start_kitchen() arranca el toolkit con la cocina custom (monkeypatch).
phase1_lib.py   : aim_head() = Fase 1a. Flags do_base_orient/do_approach (en el agarre
                  se llama con do_approach=False: solo busca/centra, sin ir recto).
fase1.py        : entry Fase 1a. "uv run Pablo/fase1.py huevo [ver]".
positioning.py  : nav/posicionamiento. Incluye:
                  - localize_with_head_camera (POR CAMARA: profundidad + back-projection)
                  - remember_object (oraculo, solo para VERIFICAR el error)
                  - compute_grasp_xy, face_arm_at_object (gira para apuntar el brazo)
                  - nav_to_parallel  <-- acercamiento PARALELO + esquive LiDAR + control
                    de base con PISO MINIMO de comando (clave del fix de congelamiento)
                  - turn_to, drive_forward, stop_base, lidar_front_min, _base_pose, _wrist_xy
grasp_lib.py    : Fase 2 (AGARRE). position_for_grasp(...method=) deja al robot ARRIBA +
                  PARALELO + apuntando + RECOGIDO. grasp_object(...method=) despacha:
                  * "lateral" (default, recomendado): muneca HORIZONTAL, baja a la altura
                    del objeto RECOGIDO (gripper sobre el pasillo, sin chocar), extiende
                    el brazo y CIERRA usando la PROFUNDIDAD central de la muneca.
                  * "top" (desde arriba): muneca abajo; lleva el CENTRO DE AGARRE
                    (link_grasp_center, cinematica) SOBRE el objeto -> extiende arm_out
                    (radial, preciso) ALTO, BAJA a la altura del objeto, CIERRA firme.
                  Helpers: _gripper_z, _grasp_center (camara d405 fresca + offset
                  calibrado GC_RADIAL=-0.05, GC_DZ=-0.174), _base_pulse, _close_and_lift.
fase2.py        : entry AGARRE. "uv run Pablo/fase2.py huevo [top] [ver]" (default lateral).
fase1b.py       : entry Fase 1b (localiza por camara + altura segura + wrist).
posicionar.py   : entry posicionamiento.
_test_nav.py    : prueba aislada de nav_to_parallel (imprime lateral/along, sin choque).
_diag_base.py   : diagnostico de velocidad de la base.
_diag_gripper.py: imprime el offset camara d405 -> link_grasp_center (calibracion).

HECHO Y VERIFICADO:
  [x] Entorno Windows completo (render, sim, cocina + 4 camaras).
  [x] Escena custom: 5 objetos garantizados, estables, mostrador limpio.
  [x] Deteccion oraculo (~2 px). Localizacion POR CAMARA (~3 cm vs verdad).
  [x] FIX bug multi-junta (move_to_many) + PosServo.
  [x] Fase 1a (cabeza): buscar + acercarse + centrar.
  [x] ALTURA SEGURA: sube el gripper por ENCIMA del mostrador antes de extender/rotar.
  [x] ACERCAMIENTO PARALELO con esquive reactivo (nav_to_parallel). VERIFICADO en
      _test_nav.py: llega paralelo, alcanzable (lateral ~0.5-0.6 m), sin choque.
  [x] *** FIX SLUGGISHNESS/CONGELAMIENTO DE LA BASE *** (era el bloqueo #1):
      el sim tiene FRICCION ESTATICA -> con comando de base CHICO la base se CONGELA.
      Solucion: en los controladores de base, los comandos no-cero van por encima de un
      PISO MINIMO decisivo (giro >=0.45, avance >=0.35) para vencer la friccion, y se
      frena por distancia para no sobrepasar. Verificado 4/4 corridas: la nav llega
      consistente, sin congelarse.
  [x] AGARRE REESTRUCTURADO (sin choques ni barridos):
      - Lateral: ya no choca (baja RECOGIDO sobre el pasillo y extiende; cierra por
        profundidad central de la muneca, sin depender del RGB de la muneca).
      - Desde arriba: lleva el CENTRO DE AGARRE (cinematica, link_grasp_center via la
        pose FRESCA de la camara d405 + offset calibrado) SOBRE el objeto. Geometria
        RESUELTA: en buenas corridas el centro de agarre queda a radial 4 mm / a-lo-largo
        11 mm / dz 29 mm del objeto, y el huevo se levanta. Lecturas basura de pose
        (pull_status a veces stale) FILTRADAS con un guard.

LIMITACION ACTUAL (lo unico que impide un agarre 100% confiable):
  [!] EJE A LO LARGO DEL MOSTRADOR. Poner el gripper sobre el objeto necesita 2 ejes:
      RADIAL (arm_out, PRECISO ~1 cm) y A-LO-LARGO del mostrador. El eje a-lo-largo SOLO
      lo puede dar la BASE (el brazo solo telescopia radial), y la base del sim NO hace
      ajuste FINO: con comando chico se CONGELA (friccion) y con comando decisivo
      SOBREPASA, y al girar/trasladar DERIVA (corrompe la geometria si se mueve con el
      brazo fuera -> ademas BARRE y tira los objetos del mostrador). face_arm (giro en
      sitio) deja un residuo a-lo-largo VARIABLE ~1-9 cm. Cuando el residuo es < ~4 cm
      el objeto entra en el span de los dedos (~9 cm) y se agarra; cuando es ~9 cm,
      queda fuera y falla. => El agarre funciona de forma INTERMITENTE, limitado por la
      PRECISION DE LA BASE del sim (no por la vision ni por la cinematica del brazo).

  Nota geometrica importante: un agarre PURAMENTE lateral (dedos colgando, brazo
  horizontal a la altura del objeto) CHOCA contra el mostrador porque los objetos
  sobresalen poco (~4 cm) y los dedos (~17 cm) golpean la superficie. Por eso, para
  objetos sobre superficie plana, el metodo DESDE ARRIBA es el adecuado; el lateral
  encaja mejor para objetos que sobresalen / al borde.

PENDIENTE:
  [ ] Hacer el eje a-lo-largo CONFIABLE. Opciones: (a) servo de base por ODOMETRIA fino
      con pulsos calibrados + re-face tras la deriva (RECOGIDO+ALTO para no barrer);
      (b) reducir la VARIANZA de face_arm; (c) en hardware real / WSL la base puede ser
      mas precisa y el problema desaparece.
  [ ] Fase 3: llevar a la estufa + abrir cajones (hay 5 joints "slide" = cajones en
      layout 0). Subir brazo antes de rotar; anti-colision LiDAR.
  [ ] (Opcional) YOLO real ("detector":"yolo") + auto-etiquetado con el oraculo.

6) HALLAZGOS TECNICOS CLAVE
--------------------------------------------------------------------------------
- BASE: la lentitud/congelamiento era FRICCION ESTATICA del sim, no la rampa ni PosServo.
  Fix = PISO MINIMO de comando en los controladores de base (nav_to_parallel). La base
  sigue SIN servir para ajuste FINO (cm): se congela chico / sobrepasa decisivo / deriva
  al girar. Mover la base con el BRAZO EXTENDIDO sobre el mostrador BARRE y tira objetos.
- AGARRE: localizar el objeto por CAMARA (cabeza) y conocer el GRIPPER por CINEMATICA es
  lo correcto y suficiente para 2 de 3 grados (radial+altura). _grasp_center se calcula
  con la pose FRESCA de la camara d405 (la del server a veces sale STALE -> guard que
  descarta z implausibles). Offset camara->centro de agarre (muneca abajo): radial -0.05,
  vertical -0.174 (medido con _diag_gripper.py).
- Camara MUNECA (d405): RGB POCO FIABLE para centrar a la pose de agarre (oclusion de los
  dedos + FOV recortado). La PROFUNDIDAD central (region fija, sin depender del RGB) SI es
  fiable -> se usa para el cierre del lateral.
- Camara CABEZA (d435i): portrait (rot90(-1)); en el servo de imagen PAN->ey, TILT->ex
  (Jacobiano online Broyden). Back-proj localizacion: Xc=(u-cx)z/f, Yc=-(v-cy)z/f, Zc=-z;
  world=cam_pos+R@[Xc,Yc,Zc]. pixel mostrado->nativo (head): u=disp_y, v=(H_nat-1)-disp_x.
- Cinematica: el BRAZO se extiende LATERAL hacia base -Y_local; gripper ~0.34 m de la base
  retraido. -Y_local=(sin th,-cos th) (radial); x_local=(cos th,sin th) (a-lo-largo).
  LiDAR: frente = indice 94 (360 rayos).
- Geometria layout0: mostrador en x; objetos y~-0.35; pasillo y mas baja; estufa x>2.8.
